HalftoneRenderer

HalftoneRenderer class

Loads the halftone FragmentPrograms once and produces configured shaders / baked images from a HalftoneGradient.

This is a pure dart:ui utility: build a shader with shaderFor / shaderForPathDots and apply it as a Paint.shader inside your own component's render(canvas), or bake to a cached Image for the blit-while-scrolling path. See docs/halftone.md.

Methods

bake(HalftoneGradient gradient, Path path, {bool clip = true, double? feather, Image? lut}) → Future<HalftoneBake>
Renders gradient, filling path, into an offscreen image. This is the production path: bake a slice once, then blit the image while it scrolls. The output auto-fits path's bounds and HalftoneBake.origin gives the placement offset. For a Sizzle StrokePath, pass its toPath(); to fill a plain rectangle, pass a rectangular path.
buildLut(HalftoneGradient gradient) → Future<Image>
Builds the amount LUT for gradient (its stops + growth mode). Callers that bake or paint many frames from the same stops can build this once and pass it back in via shaderFor / bake to avoid rebuilding it.

Configures a path-dots FragmentShader: like shaderFor, but each dot is kept only if its cell centre lies inside mask (from rasterizePathMask) and is then drawn whole (never clipped at the path boundary).

pathDotsSpill({required double gridSize, double? feather}) → double
The largest distance (px) a whole/feathered dot can extend past the path edge, for a given gridSize and feather (defaults to gridSize * 0.6). This is the margin a whole-dot bake (clip: false) adds when auto-fitting; expose it if you want to size/offset a bake yourself.

Constants

defaultTone → const List<HalftoneStop>
Default tone curve for halftoneImage: dark source (luminance 0) → full dot, light source (luminance 1) → no dot. Reproduces a photo as foreground ink on the background.
